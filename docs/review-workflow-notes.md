# Review workflow notes

Use these notes to keep small reviews consistent and easy to audit.

## Before review

- Confirm that the pull request has a clear title.
- Confirm that the pull request describes the scope.
- Confirm that the change is small enough to review.
- Confirm that no unrelated files were changed.

## During review

- Read the changed files before approving.
- Check that documentation changes are accurate and useful.
- Check that no secrets or personal data were added.
- Check that the pull request does not hide unrelated maintenance work.

## After review

- Leave the pull request open if checks are still pending.
- Merge only after the expected checks and review requirements are satisfied.
- Keep the review trail simple and understandable.
