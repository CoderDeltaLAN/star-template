# Documentation review notes

Use these notes when reviewing documentation-only pull requests.

## Clarity checks

- Confirm that the document has one clear purpose.
- Confirm that the title matches the content.
- Confirm that instructions are understandable without private context.
- Confirm that examples use safe placeholder values.

## Maintenance checks

- Confirm that the new document does not duplicate existing guidance.
- Confirm that links and file names are accurate.
- Confirm that future contributors can follow the notes.
- Confirm that the document can age well without constant edits.

## Final pass

- Review the final diff before approval.
- Keep wording direct and practical.
- Leave unclear ideas as follow-up work.
- Avoid adding process noise that does not help maintainers.
