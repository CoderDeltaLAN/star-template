# Review trace notes

Use these notes to keep a small pull request review easy to trace.

## Trace points

- Confirm the branch name matches the change.
- Confirm the pull request title matches the diff.
- Confirm the changed file is expected.
- Confirm the review scope is clear.

## Evidence

- Keep the change documentation only when stated.
- Keep commit messages aligned with the content.
- Keep private context out of repository files.
- Keep secrets and personal data out of the diff.

## Closeout

- Confirm that checks completed successfully.
- Confirm that the pull request was reviewed.
- Confirm that the merge used the expected head commit.
- Leave follow-up work for a separate branch.
