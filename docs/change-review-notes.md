# Change review notes

Use these notes before requesting review for a small repository change.

## Review scope

- Confirm that the change has one clear purpose.
- Confirm that only expected files were edited.
- Confirm that the pull request title matches the diff.
- Confirm that follow-up ideas are left for another branch.

## Diff quality

- Prefer short documentation updates.
- Avoid adding private chat context.
- Avoid adding local machine details.
- Avoid mixing unrelated maintenance tasks.

## Reviewer handoff

- State what changed.
- State what did not change.
- State whether secrets or personal data were avoided.
- Leave the branch ready for a normal review.
