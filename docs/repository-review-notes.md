# Repository review notes

Use these notes when doing a quick repository maintenance review.

## Repository state

- Confirm that the main branch is current.
- Confirm that open pull requests are intentional.
- Confirm that recent merged pull requests have clear titles.
- Confirm that documentation changes are easy to understand.

## Review focus

- Look for stale branches that may need cleanup later.
- Look for documentation that duplicates existing files.
- Look for instructions that are too local to one machine.
- Look for process notes that future contributors can follow.

## Follow-up

- Keep follow-up changes in separate branches.
- Avoid mixing repository review with feature work.
- Record unclear items instead of guessing.
- Leave the repository easier to audit than before.
