# Branch cleanup notes

Use these notes after a small pull request has been merged.

## Cleanup steps

- Confirm that the pull request is merged.
- Confirm that the main branch includes the merged change.
- Remove local branches only after the merge is verified.
- Remove remote branches only when they are no longer needed.
- Keep forks available until the collaboration history is no longer being reviewed.

## Review reminders

- Do not delete branches that still have open pull requests.
- Do not delete forks while achievement or contribution tracking is still settling.
- Keep cleanup separate from feature work.
- Leave a clear repository history for future maintainers.
