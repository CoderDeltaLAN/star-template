# Maintenance notes

Use these notes when making small maintenance changes to this template.

## Maintenance habits

- Keep maintenance changes focused.
- Prefer one purpose per pull request.
- Review generated files before committing.
- Avoid unrelated formatting changes.
- Confirm that documentation still matches the current project structure.

## Review reminders

- Check the changed file list.
- Confirm that no secrets, tokens, logs or private data were added.
- Keep the pull request body clear and short.
- Leave larger refactors for a separate branch.
