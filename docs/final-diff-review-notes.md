# Final diff review notes

Use these notes before sending a small documentation change for review.

## Diff scope

- Confirm that the diff contains only expected files.
- Confirm that the file name matches the purpose.
- Confirm that the change is small enough to review.
- Confirm that unrelated cleanup was not included.

## Content check

- Read the added text once before pushing.
- Check that wording is clear and direct.
- Check that no private context was copied.
- Check that no secrets or personal data were added.

## Review handoff

- Keep the pull request title short.
- Keep the body focused on scope.
- Mention that the change is documentation only.
- Leave follow-up work for a separate pull request.
